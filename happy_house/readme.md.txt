# 상휘민정 마통 & 중도금 관리 페이지

## 만들고 싶은 것

GitHub Pages로 배포되는 개인용 웹 페이지입니다.

예시 URL:

https://mjson0212.github.io/my_fist_web/happy_house/

이 URL에 접속하면 별도 로그인 없이 두 사람이 같은 데이터를 보고 입력할 수 있습니다.

목적은 마이너스통장에 돈이 얼마나 모였는지, 출금 가능 금액이 얼마나 남았는지, 5회차/6회차 중도금까지 얼마나 부족한지를 쉽게 확인하는 것입니다.

---

## 주요 기능

### 1. 입금 / 출금 내역 입력

사용자는 날짜별로 아래 내용을 입력할 수 있습니다.

- 날짜
- 구분
  - 입금
  - 출금
  - 중도금 납부
- 금액
- 메모

예시:

| 날짜 | 구분 | 금액 | 메모 |
|---|---:|---:|---|
| 2026-05-10 | 입금 | 5,000,000 | 월 저축 |
| 2026-06-01 | 출금 | 1,000,000 | 생활비 사용 |
| 2027-12-01 | 중도금 납부 | 75,000,000 | 5회차 중도금 |

---

### 2. 설정 탭

설정 탭에는 자주 바뀌지는 않지만 기준이 되는 값을 입력합니다.

- 총 마이너스통장 한도
- 현재 출금 가능 금액
- 총 받은 대출액
- 5회차 중도금 필요 금액
- 6회차 중도금 필요 금액
- 5회차 납부 예정일
- 6회차 납부 예정일
- 월 목표 저축액

예시:

| 항목 | 값 |
|---|---:|
| 마통 한도 | 80,000,000 |
| 현재 출금 가능 금액 | 20,000,000 |
| 5회차 필요 금액 | 75,000,000 |
| 6회차 필요 금액 | 75,000,000 |
| 월 목표 저축액 | 5,000,000 |

---

## 계산 방식

### 현재 모은 금액

출금 가능 금액을 기준으로 계산합니다.

현재 모은 금액 = 현재 출금 가능 금액 - 초기 출금 가능 금액 + 입금 합계 - 출금 합계

다만 중도금을 납부하면 출금 가능 금액이 크게 줄어들기 때문에, 회차별 목표는 분리해서 관리합니다.

---

## 목표 달성 상태

5회차와 6회차를 각각 따로 보여줍니다.

### 5회차

- 5회차 필요 금액
- 현재 5회차 기준으로 모은 금액
- 부족 금액
- 납부일까지 남은 개월 수
- 월별 필요 저축액
- 달성률 게이지

### 6회차

5회차를 납부한 이후에도 이어서 볼 수 있도록 별도 목표로 관리합니다.

- 6회차 필요 금액
- 현재 6회차 기준으로 모은 금액
- 부족 금액
- 납부일까지 남은 개월 수
- 월별 필요 저축액
- 달성률 게이지

## 화면 구성

### 대시보드

첫 화면에서는 아래 내용을 한눈에 보여줍니다.

- 현재 출금 가능 금액
- 5회차 중도금 달성률
- 6회차 중도금 달성률
- 전체 부족 금액
- 남은 개월 수
- 월별 필요 저축액

### 입출금 내역 탭

- 입금/출금/중도금 납부 내역 입력
- 내역 수정
- 내역 삭제
- 월별 합계 확인

### 설정 탭

- 마통 한도
- 초기 출금 가능 금액
- 현재 출금 가능 금액
- 5회차 필요 금액
- 6회차 필요 금액
- 납부 예정일 수정

## 고민 중인 부분

### 중도금 납부 후 출금 가능 금액 초기화 여부

5회차 중도금을 납부하면 출금 가능 금액이 크게 줄어듭니다.

이때 전체 금액을 단순 누적하면 헷갈릴 수 있으므로, 아래 방식이 좋습니다.

- 5회차 목표와 6회차 목표를 분리한다.
- 5회차 납부 완료 시, 5회차 목표는 `완료` 상태로 표시한다.
- 이후부터는 6회차 목표 기준으로 게이지를 계산한다.
- 출금 가능 금액은 실제 마통 상태 그대로 입력한다.
- 목표 달성 계산은 `회차별 목표` 기준으로 따로 관리한다.

즉, 출금 가능 금액을 강제로 초기화하기보다는  
“5회차 완료 처리”를 하고 6회차 계산으로 넘어가는 방식이 더 안전합니다.

## 데이터 저장 방식

Firebase Firestore를 사용합니다.

로그인은 사용하지 않고, 방 코드 또는 문서 ID 기준으로 데이터를 저장합니다.

예시 컬렉션:

happy_house_sync

예시 문서:

happy_house_sync/{방코드}

방 코드는 16자리 이상으로 입력합니다.

---

## Firebase 규칙 예시

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /happy_house_sync/{docId} {
      allow read, write: if docId.size() >= 16;
    }
  }
}
